# 06 — 90-Day Build Roadmap

Assumption: nights/weekends solo build alongside day job (proven cadence). Target: closed beta by Day 90; public launch ~Dec 2026–Jan 2027 (6–8 weeks before Ramadan 2027).

## Phase 1 — Foundation (Days 1–21)

- [ ] Name decision + trademark/App Store/domain check; reserve bundle IDs
- [ ] Monorepo scaffold (pnpm workspaces, Expo dev client, Next.js web)
- [ ] `prayer-core` package: adhan-js wrapper, all calc methods incl. Diyanet
- [ ] **Golden test suite**: 20+ cities × 4 seasons vs published Diyanet/ISNA/London Unified/MWL tables (Vancouver + Oslo June cases mandatory)
- [ ] High-latitude fallback logic + auto-suggestion by region
- [ ] Design system basics (packages/ui) — calm, premium, uncluttered; no green-and-gold clichés unless intentional
- [ ] Landing page live with waitlist (start collecting emails NOW)

**Exit criteria:** calc engine passes golden tests; waitlist collecting.

## Phase 2 — The Notification Engine (Days 22–45)

The make-or-break phase. Budget the most time here.

- [ ] iOS: scheduling window manager (64-slot budget, reschedule triggers), athan sound strategy decided (chained segments vs open-app playback) and implemented
- [ ] Android: exact alarms + foreground-service athan playback + full-screen intent
- [ ] Vendor battery-whitelist flows (Samsung/Pixel/Xiaomi deep links, device-detected copy)
- [ ] **Notification Health Check** screen (test-fire, risk detection, guided fixes)
- [ ] Settings persistence layer + automated tests (Pillars' stuck-settings bug = our regression test)
- [ ] On-device delivery ledger + diagnostic export
- [ ] E2E tests (Maestro): reboot, DST change, timezone change, battery saver on

**Exit criteria:** 7 consecutive days of 100% on-time athan delivery on Pixel + Samsung (battery saver ON) + iPhone.

## Phase 3 — Product Surface (Days 46–70)

- [ ] Today screen (countdown, 5 times, sunrise, Hijri)
- [ ] Qibla compass (no-movement init)
- [ ] Prayer tracker + streaks (one-tap logging, gentle tone)
- [ ] Travel mode (change detection, prompt, qasr guidance, revert)
- [ ] iOS widgets (next-prayer, all-times WITH sunrise) + lock screen
- [ ] Hijri calendar + offset setting
- [ ] Onboarding: location → method auto-suggest → Health Check → done in <90s
- [ ] Mosque timetable: pin mosque, manual timetable entry, offline cache, "compare with my mosque" view (Supabase minimal backend)

**Exit criteria:** daily-drivable app; founder + wife using it as only prayer app for 1 week.

## Phase 4 — Beta & Hardening (Days 71–90)

- [ ] Public roadmap page + voting live; link in-app
- [ ] Privacy page: plain-language + "every network request we make" list
- [ ] Sentry (PII-scrubbed) + opt-in self-hosted analytics
- [ ] TestFlight + Play closed track; recruit 30–50 testers via local mosque WhatsApp groups, r/islam beta thread, Turkish community contacts
- [ ] Weekly beta builds; fix top delivery-reliability issues from diagnostic reports
- [ ] Store listings drafted: screenshots, ASO keywords ("prayer times", "athan", "qibla", "namaz vakitleri" for TR later)
- [ ] Review-response pipeline in n8n ready (adapt LoyalLocal Review Responder)

**Exit criteria:** beta cohort reporting ≥99% notification reliability; store assets ready.

## Post-90 (Launch runway: Days 91–150)

- v1.0 public launch (Dec 2026 target) → App Store featuring pitch (privacy + Ramadan angle)
- v1.1: Quran reader, Apple Watch complications, Turkish localization + Diyanet default for TR users
- **F13 Ramadan Mode ships by mid-Jan 2027**: suhoor/iftar Live Activity, fasting tracker, Ramadan dashboard
- RevenueCat + Plus tier ships with v1.1 (launch free-only first — earn reviews before asking for money)
- Ramadan 2027 (~Feb 17): full marketing push (see 07)

## Weekly operating rhythm

- Mon–Thu evenings: build
- Fri: beta feedback triage + review responses
- Sat: deep-work block (hardest engineering)
- Sun: content batch for launch channels (n8n-scheduled)

## Kill / pivot criteria (honest checkpoints)

- Day 45: if notification reliability can't hit 95%+ on Android matrix → pause features, fix or rethink (this IS the product)
- Day 90: if beta D7 retention < 25% → diagnose before spending on launch
- Ramadan+30: if conversion < 1% with 50K+ downloads → revisit premium bundle composition
